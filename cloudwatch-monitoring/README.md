# Amazon CloudWatch Monitoring

## Objective
Monitor AWS resources and understand operational visibility using Amazon CloudWatch.

## Services Used
- Amazon CloudWatch
- Amazon EC2 (metrics only)

## What I Did
- Reviewed EC2 performance metrics
- Created CloudWatch alarms based on thresholds
- Differentiated between logs, metrics, and alarms
- Analyzed resource utilization patterns

## Key Concepts Learned
- Cloud monitoring fundamentals
- Proactive alerting
- Operational awareness in cloud environments

## Outcome
Established foundational monitoring and alerting practices using AWS-native tools.

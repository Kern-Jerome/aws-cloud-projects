# AWS Identity and Access Management (IAM)

## Objective
Implement identity and access controls using AWS IAM following least-privilege principles.

## Services Used
- AWS IAM

## What I Did
- Created IAM users and groups
- Assigned managed policies based on role responsibilities
- Evaluated permission scope and access boundaries
- Removed users after validation to maintain security

## Key Concepts Learned
- Least privilege access
- IAM users vs groups vs roles
- Cloud security fundamentals

## Outcome
Demonstrated secure access control implementation within an AWS environment.
